## 1. 設計理念與教學定位

### 1.1 核心理念
**「以歷史現場代入感，取代死背年表」**——讓學生扮演晚清官員身邊的「書記」，透過收集線索、推理、撰寫奏摺，內化 DSE 所要求的「背景—措施—評價」三層分析能力。

### 1.2 教學目標（對應 DSE 評卷）
1. **史實掌握**：人物、年份、機構、條約、戰役。
2. **因果分析**：內憂外患 → 自強動機 → 措施 → 成敗原因。
3. **史料運用**：分辨史料類型，引用支撐論點。
4. **論證寫作**：透過「奏摺工作坊」訓練段落結構（論點 + 史料 + 評價）。

### 1.3 設計原則
- **不直接給答案**：玩家須自行從場景物件中發掘線索。
- **沉浸優先**：避免突兀的「教科書旁白」；改以 NPC 對話、私函、奏摺等歷史文體呈現知識。
- **可重玩**：決策回響、論點句變體、回憶錄成就，鼓勵多周目。
- **零外部依賴**：音效用 Web Audio API 程序生成；圖像為背景靜圖。

## 2. 玩家身份框架：書記（Secretary）

### 2.1 為何選書記？
- **解決時代錯置**：玩家不可能與 1842 年的耆英、1895 年的李鴻章同時對話。書記是匿名隨員，可跨章節轉任不同官員。
- **教學優勢**：書記負責「整理卷宗、草擬文書」，天然對應「收集史料、撰寫奏摺」玩法。
- **情感連結**：書記在章節之間寫私函予同窗，反思時局，作為知識回顧與情感緩衝。

### 2.2 書記履歷
| 章節 | 年份 | 服侍官員 | 地點 |
|------|------|----------|------|
| Ch1  | 1839–1860 | 耆英 / 奕訢 | 廣州 / 北京 |
| Ch2  | 1861–1872 | 曾國藩 / 李鴻章 / 左宗棠 | 安慶 / 上海 / 福州 |
| Ch3  | 1872–1885 | 李鴻章 / 沈葆楨 / 盛宣懷 | 天津 / 上海 |
| Ch4  | 1885–1895 | 張之洞 / 李鴻章 | 武漢 / 威海 |

### 2.3 私函系統（Secretary Letters）
3 封私函穿插於章節之間，書信對象為同窗「子敬兄」。每封結尾附 **複習問答**（spaced repetition），白話文撰寫以利學生理解。

## 3. 遊戲整體結構

```
LANDING（封面）
 ├─ 開始新局 → CHAPTER 1
 ├─ 翻閱回憶錄（已通關後解鎖）
 └─ 載入存檔
       │
       ▼
CHAPTER 1（1839–1860 外患內憂）
  ├─ Scene 1_1：虎門焚煙後
  ├─ Scene 1_2：南京條約現場
  ├─ Scene 1_3：英法聯軍與京師震動
  └─ 工作坊 1：論「自強之必要」
       ↓ 私函 1
CHAPTER 2（1861–1872 求強）
  ├─ Scene 2_1：安慶內軍械所
  ├─ Scene 2_2：江南製造總局
  ├─ Scene 2_3：福州船政局
  ├─ Scene 2_4：同文館
  └─ 工作坊 2：論「求強措施得失」
       ↓ 私函 2
CHAPTER 3（1872–1885 求富）
  ├─ Scene 3_1：輪船招商局
  ├─ Scene 3_2：開平煤礦
  ├─ Scene 3_3：上海機器織布局
  ├─ Scene 3_4：電報與鐵路
  └─ 工作坊 3：論「求富之轉向」
       ↓ 私函 3
CHAPTER 4（1885–1895 高峰與崩潰）
  ├─ Scene 4_1：北洋艦隊
  ├─ Scene 4_2：漢陽鐵廠
  ├─ Scene 4_3：甲午海戰與馬關
  └─ 工作坊 4：論「洋務運動失敗原因」
       ↓
ENDING（終局結算）→ 回憶錄歸檔 + 成就解鎖
```

### 3.1 每場景流程
1. **進場敘述**：書記視角開場（1–2 段）
2. **場景探索**：點擊背景物件收集線索（evidence）
3. **NPC 對話**：與官員對話，部分對話需推理選擇
4. **推理題**：基於線索作出判斷（含「下一步」手動推進）
5. **說服橋段**：選擇 **單一** 最具說服力線索（不可多選、不可沉默）
6. **場景結算**：解鎖卷宗，更新 timeline

### 3.2 工作坊四階段（每章末一次）
1. **揀選卷宗**：從已收集線索中選 3–5 條
2. **論點生成**：每條線索對應一個論點，分屬不同 aspect
3. **奏摺成稿**：自動組裝為段落（論點句 + 史料引用 + 評價句）
4. **歷史回響**：對照真實歷史結局，給予評分

## 4. 核心系統規格

### 4.1 線索系統（Evidence）
- 每條 evidence 屬性：
  ```js
  {
    id: 'e_xxx',
    title: '虎門炮台殘垣',
    type: '實物 / 文獻 / 口述 / 圖像',
    sceneId: 's1_1',
    aspect: 'military_gap',   // 對應工作坊論點分類
    summary: '...',
    detail: '...（DSE 標準史料說明）',
    workshopExclude: false,   // 若為 true 則不進入工作坊
    isPuzzleTarget: false,
  }
  ```
- **獲取方式**：點擊場景熱點 / 對話選項解鎖 / 推理正確獎勵。

### 4.2 推理系統（Inference）
- 每場景 1–2 題單選題。
- 題幹基於已收集線索，選項含 1 正解 + 2–3 干擾項（DSE 常見誤區）。
- 答錯不阻擋進度，但會累加 `inferenceStats.wrong`，影響成就。
- 答對 / 答錯後均顯示「下一步」按鈕，玩家自行掌握節奏。

### 4.3 說服系統（Persuasion）
- **單選**：玩家從已解鎖 evidence 中選 **一條** 作為說服 NPC 的證據。
- **無沉默選項**：必須選擇。
- 系統根據 evidence 的 aspect 與場景目標 aspect 匹配，給予 3 級回饋（強 / 中 / 弱）。
- 選擇被記錄於 `state.persuasionChoices`，供後續決策回響引用。

### 4.4 工作坊系統（Memorial Workshop）
- **Phase 1 揀選卷宗**：以 aspect 分組顯示，需覆蓋 ≥3 aspects（若該章只有 2 aspects 則 fallback 至 2）。
- **Phase 2 論點生成**：呼叫 `buildArgumentSentence(point, chapterKey)` 生成論點句。
  - 句式：「[開頭歷史脈絡句] + [史料引用：xxx 顯示...] + [評價：故...]」
  - **變體機制（Phase 2.3, 計劃中）**：每 point 提供 2–3 句變體，依 `memoirRunCount` hash 選擇，避免機械記憶。
- **Phase 3 奏摺成稿**：自動組裝為仿奏摺體，可列印 PDF。
- **Phase 4 歷史回響**：對照真實結局段落（如「然北洋艦隊終敗於甲午…」）。

### 4.5 決策回響（Decision Echoes）
跨章節呼應玩家早期選擇。已實現映射：
| 來源場景 | 目標場景 | 回響內容 |
|----------|----------|----------|
| s1_3 說服選擇 | s2_2 對話 | 「上次你提及…，今果如所言」 |
| s2_2 說服選擇 | s3_3 對話 | 影響盛宣懷對官督商辦的態度 |
| s3_1 說服選擇 | s4_2 對話 | 影響張之洞鐵廠決策對白 |
| s1_2 揀卷選擇 | s4_3 奏摺評語 | 影響終局評分微調 |

### 4.6 成就系統（Achievements）
14 項，5 大類：

| 類別 | 成就示例 |
|------|----------|
| 進度（Progress） | 初任書記、四章功成 |
| 史料（Evidence） | 線索達人（解鎖全部 evidence）、文獻通、實物鑑 |
| 決策（Decision） | 鐵腕派、實業派、外交派 |
| 奏摺（Memorial） | 字字珠璣（4 章工作坊滿分）、墨痕千里（列印一次） |
| 特殊（Special） | 一週目通關、無錯推理、見證者（解鎖全部回響） |

觸發於：`onQuizContinue` / `wsGotoPhase(4)` / `showEnding` / 列印按鈕。
顯示：右下角 toast，2.5s 自動消失，附音效。

### 4.7 回憶錄系統（Memoir Archive）
- localStorage key：`ziqiang_memoirs_v1`
- 每次通關 `archiveRun()` 寫入：時間戳、各章工作坊摘要、解鎖成就、總分。
- 入口：封面右上「翻閱回憶錄」。
- 頁面：成就徽章網格 + 過往通關卡片列表 + 重玩按鈕。

### 4.8 首次玩家引導（Onboarding）
- localStorage key：`ziqiang_onboarding_v1`
- 3 個觸發點：`first_scene`（首次進場景）/ `first_inference`（首次推理題）/ `first_workshop`（首次工作坊）
- 形式：浮動 tooltip，1 次性顯示，可手動關閉。

### 4.9 場景進度指示
- timeline-info 區域顯示 `場景 N / 總場景數`（如「場景 7 / 14」）
- 每章進場時更新

## 7. UI / UX 與美學規範

### 7.1 紙質美學（Paper Aesthetic）
- CSS 變數：
  ```css
  --paper-bg: #f3e6c6;
  --paper-fiber: url('...noise-fiber.svg');
  --paper-stain: radial-gradient(...茶漬...);
  --ink-black: #2a1f17;
  --seal-red: #a8322a;
  ```
- 字體：標題用「思源宋體 / 仿宋」毛筆風；正文用「思源黑體」。
- 印章：紅色篆書印（SVG），用於章節標題、奏摺末尾。
- 紙頁邊緣：CSS `mask-image` 模擬撕邊。

### 7.2 環境音效（Web Audio API）
全部程序生成，無外部檔案：
- **翻頁聲**：white noise + 高通濾波 80ms
- **印章蓋落**：低頻撞擊 + 短殘響
- **筆觸**：頻率掃描 200–400Hz
- **成就解鎖**：和諧 chord（C-E-G）短促
- **背景氛圍**：低音量風聲 / 蟬鳴（依章節切換）

### 7.3 互動原則
- 所有可點擊熱點懸停顯示放大鏡 cursor
- 推理題答錯不立即推進，允許玩家回看線索
- 工作坊每階段可返回上一步

### 7.4 列印 / PDF
- 終局奏摺與通關回憶錄均提供「列印」按鈕
- 使用瀏覽器原生 `window.print()` + 列印專用 CSS

## 10. DSE 涵蓋率與補強策略

### 10.1 涵蓋率（補強前 82% → 補強後 97%）
**補強方式**（不新增 evidence，避免膨脹）：
- **A. NPC 對話內嵌**：將原本散見於教科書的概念融入官員對白。
- **C. 私函補述**：3 封書記私函以白話文補強：
  - **Letter 1**（Ch1 末）：八里橋之敗、奕訢留京、虎門炮台殘垣感想
